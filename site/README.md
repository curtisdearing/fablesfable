# fablesfable — transparency site

A static, deployable front end that makes the prop model a **glass box**: every
pick traces back through its projection, the measured adjustments applied to it,
the model-vs-market comparison, and the tier decision — so you can see *why* one
bet is more attractive than another, not just *that* it is.

## What's here

```
site/
  index.html        single-page app (tabs: Picks · How it works · Model · Constants · Evidence)
  assets/app.js     renders everything from data/*.json (no framework, no build)
  assets/style.css
  data/*.json        generated by scripts/export_site.py  (git-ignored; regenerate)
```

Five views:

- **Picks & why** — the week's picks grouped by game. Each is tiered
  (STRONG / PLAYABLE / LEAN / RESEARCH) and expands to a **decision chain**:
  projection → adjustments (weather, injury, reallocation, game script) → market
  comparison (model prob vs de-vigged fair prob, edge, EV) → tier reason.
- **How it works** — the seven pipeline stages, in order, with the rule that
  keeps it honest: selection happens *last*, only after every candidate is scored.
- **The model** — all 67 walk-forward features, the composite weights, the fixed
  matchup sub-weights, and the per-market selector thresholds.
- **Measured constants** — every shipped constant with its fit provenance and the
  script that produced it (weather coefficients, TD blend, PROE, absence boost,
  recency weight…), plus the effects that were measured and *rejected*.
- **Evidence** — the CLV kill-check verdict, graded-pick record by tier, the
  recency-weight fit result (every market, every season), and the situational
  tags that were tested and gated out.

## Generate the data + view it

```bash
# after a weekly pipeline run (or any time — falls back to a labeled sample):
python3 scripts/export_site.py            # writes site/data/*.json

# serve it (fetch of local JSON is blocked under file://):
cd site && python3 -m http.server 8000    # then open http://localhost:8000
```

The exporter reads the warehouse DB, the JSON artifacts, and the module
constants directly — **no parquet dependency**, so it runs anywhere the package
imports. When no live week exists it emits a clearly-labeled **sample** built by
running the real selector on illustrative inputs (`scripts/make_sample_picks.py`).

## Deploy

It's fully static. Point GitHub Pages / Netlify / Cloudflare Pages at the
`site/` directory (or copy it to any web root). Re-run the exporter and
redeploy after each weekly pipeline pass — a CI step can do both.

**Leans, not locks.** Research/advisory only, no bet placement; graded at
synthetic reference lines until live CLV accrues. 1-800-GAMBLER.
