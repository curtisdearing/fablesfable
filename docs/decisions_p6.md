## 6.1 Matchup granularity

- **Depth/location defense shapes** (`_build_def_shape`): per-defense walk-
  forward yards-per-target factors on short (<8 air yards) vs downfield, and
  middle vs outside (`pass_location`), normalized-by-league-mix and blended by
  the PLAYER's own trailing profile into `opp_factor` (`projection.shape_tilts`,
  each tilt capped ±10%). Receiving markets tilt on depth+location; QB passing
  on throw depth. Built on the full defense-week grid so row-missingness can
  never encode current-week info.
- **Archetype-aware shrinkage priors**: `RB_receiving`/`RB_early_down` (trailing
  target mix ≥ .35), `WR_deep`/`WR_short` (trailing aDOT split at 11.0),
  `generic` for QB/TE (TE inline-vs-move is alignment data — see paywall
  boundary) and <3 trailing games. Prior tier 1 = (role, archetype) expanding
  prior-weeks mean, tier 2 fallback = coarse role (exact pre-6.1 behavior).
  Semantic note: cold-start rows now regress toward the (role, generic) pool —
  effectively "replacement level" rather than all-role average.
- **Red-zone defense**: TDs-allowed-per-RZ-trip factor per defense-week
  (`roll_rz_td_factor`), stamped on candidates as `opp_rz_td_factor`.
- **Composite matchup weighting PINNED**: `MATCHUP_SUB_WEIGHTS = opp .25 /
  script .20 / pace .20 / epa .20 / opp_absence .15` (absence reserved for
  6.5, neutral until then). Missing dimensions contribute neutral 0.5 — the
  silent 1/3↔1/4 renormalization is gone. Components expose every sub-score
  and the weight dict.
- **Live-mode bug found & fixed**: `carry_forward` (live) weeks have no pbp
  rows, so exact `(season, week)` lookups returned None and every live
  projection ran with NEUTRAL opp/team factors — backtest and live disagreed
  silently. `WeekInputs` now serves as-of (strictly-before) rows for live
  weeks; backtest path byte-identical.

Composite-path ablations (2024 hit/top-1 | 2025 hit/top-1; SE ≈ ±1.3pt/±3pt):
baseline 59.3/61.8 | 57.1/58.5 → 6.1-full 58.7/62.1 | 57.4/59.9; tilts-only
58.5/62.9 | 57.7/59.2; archetypes-only 59.3/61.0 | 57.2/60.3. All inside
noise on this path — 6.1's measurable value shows up in the ML layer (below).

## 6.2 Red-zone / TD wiring

- `player_week` gains full-game rz/gl usage actuals + 16-game-window shrunken
  shares (`roll_rz_tgt_share`, `roll_rz_carry_share`, `roll_gl_carry_share`);
  `team_week` gains RZ opportunity volumes + PRIOR-WEEKS-ONLY league
  TD-per-RZ-opportunity series (derived walk-forward series, not constants).
- **anytime-TD lambda** = `(1-w)·(volume × trailing TD rate) + w·(team RZ
  opps × player RZ share × league TD-per-opp)`, **w = 0.40 MEASURED** by
  walk-forward log-loss grid on 2019-2023 (n=16,871 candidate-gated
  player-weeks; log-loss .5398 → .5341, Brier .1781 → .1760;
  `scripts/fit_td_blend.py`). Missing ingredients → pure base path.
- **Opponent RZ factor on lambda: measured and REJECTED** — factor-ON was
  worse at every w in the same fit. It does NOT touch the deterministic mean;
  it ships as an ML feature only (`opp_rz_td_factor`).
- **Reallocation extended to TD share**: with/without splits now measure RZ
  usage shifts too; `apply_reallocation` routes an `anytime_td` multiplier off
  the RZ-share delta (same ×1.35 cap, proportional-guess halving), so an
  RB1-out boosts the backup's TD lambda by his goal-line inheritance, not his
  snap inheritance.

## 6.3 Game script + garbage time

Both candidate mechanisms were FIT before wiring (`scripts/fit_game_script.py`,
2019-2023):

- **Opponent-pace volume term: REJECTED.** log(plays) ~ own^.292 (t=6.6) ×
  opp^(-.015) (t=-0.3), n=2,592; implied multiplier band [.9985, 1.0016].
  Opponent trailing pace adds nothing beyond own trailing volume. Not shipped.
- **Neutral-PROE split term: CLEARED.** This week's pass share ~ trailing
  share (+.416, t=10.0) + spread tilt (+.019, t=4.3) + **trailing neutral
  PROE (+.1168 per pass_oe/100, t=+3.1)**, n=2,607. Shipped: the fitted
  share-delta converts to a pass/rush volume multiplier (±3% cap = P99
  guard) inside `game_script_multipliers`; missing PROE → exact legacy
  behavior. Composite-path effect ~noise (−0.3pt), kept on the strength of
  the fit; it also feeds live weeks where spreads go stale.
- **Garbage-time filter: measured and REJECTED (ships default-OFF).**
  Definition: Q4 & (|margin| ≥ 17 or wp outside 5-95%). Filtered
  shares/efficiencies (volumes and actuals stay full-game) with prior-weeks
  league full/filtered ratio recalibration so the league level can't shift.
  Result: composite hit −1.0/−1.1pt (2024/2025), and line-free projection
  MAE a wash (receiving 24.77 vs 24.58 filtered-vs-not, receptions 1.670 vs
  1.658, passing 72.76 vs 72.60, rushing 25.52 vs 25.70). The machinery stays
  behind `GARBAGE_FILTER_ENABLED` / `build_player_week(garbage_filter=)`;
  the natural next experiment is ADDITIVE filtered columns for the GBDT
  rather than replacement.
